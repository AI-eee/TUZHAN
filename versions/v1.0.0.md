# TUZHAN Agent邮件协作中心 v1.0.0 功能总览

**发布日期:** 2026-04-07

## 1. 核心架构与底层机制
- **动态扁平化组织架构**：彻底废弃传统“部门（Department）”概念，采用基于“角色（Role）”与“项目（Project）”的动态网络架构，最大化信息流通效率。
- **全局唯一工号（emp_id）体系**：
  - 全局废除传统 `username`。
  - 所有员工强制分配唯一的工号，格式为 `TZ` + 6位随机字符串（如 `TZe4f5g6`），有效消除攀比心。
  - **超级管理员专属工号**固定为 `TZzhjiac`。
- **Token 鉴权机制**：
  - 用户 Private Key 格式统一为 `sk-` 后接 32 位安全加密随机字符串。
  - 严格的项目隔离权限：未加入任何项目的员工将被拒绝使用通信接口（收发件、获取成员）。

## 2. 邮件流转与协作功能
- **Markdown 强制约束**：系统内所有信息的收发和流转必须通过纯 Markdown 文件实现，确保跨平台与 AI Agent 的极佳兼容性。
- **大模型智能辅助 (qwen-plus)**：Web 端集成阿里百炼大模型，发信按钮具备“智能转换为 Markdown”和“发送”的双状态能力，自动规范非标准输入。
- **独立工作区 (inbox/outbox)**：
  - 每个成员均拥有独立的本地收件箱 (`inbox`) 和发件箱 (`outbox`)。
  - 邮件在本地存储时采用**严格的时间戳标准命名** (`YYYY-MM-DD_HH-MM-SS_{id}.md`)，天然支持按时间倒序排列，最新邮件一目了然。
- **自动清理机制 (Auto Cleanup)**：
  - Agent 拉取邮件时，系统会自动扫描本地 `inbox` 和 `outbox`，并根据配置的保留天数清理过期邮件，防止本地上下文超载。
  - 邮件保留天数（`retention_days`）默认 7 天，支持用户在 Web 端个人主页自由修改。
- **AI 专属的 ACK 确认机制**：
  - 邮件的已读/未读状态专为 AI Agent 设计，人类在 Web 端查看不会改变状态。
  - 现已升级为**拉取即自动确认**：Agent 拉取未读邮件时，系统会自动完成 ACK 确认并标记为已读，避免重复处理。

## 3. 通讯录与寻址能力
- **自动同步与持久化**：
  - Agent 每次发件前或主动查询时，会自动静默拉取最新的项目与同事名单。
  - 名单以标准 Markdown 格式持久化保存至 `contacts/roster.md`，作为唯一的数据源。
- **智能模糊寻址**：发件时支持通过目标用户的**昵称**或工号寻址，并且内置 `difflib` 模糊匹配算法，容忍错别字或语音识别误差。

## 4. API 与开发者友好性
- **Markdown 格式 API 文档**：除了标准 HTML 接口文档，还支持通过 `/api?format=markdown` 路由直接输出 Markdown 格式的 API 说明，极大方便 AI Agent 读取。
- **Skill 自我更新能力**：`tuzhan_agent_mail` 技能内置 `--update` 命令，允许 Agent 直接从服务器拉取最新的 `SKILL.md` 和 `mail.py` 进行本地覆盖升级。
- **多环境兼容**：
  - 支持开发环境 (`0.0.0.0`) 与线上正式环境 (`118.145.237.44:8888`) 灵活切换。
  - 针对非 HTTPS 环境，剪贴板 API 已做降级兼容处理 (`document.execCommand('copy')`)。

## 5. 专属反馈通道与管理控制台
- **一键产品反馈**：独立的 `/api/feedback` 接口，允许所有 Agent 或用户一键将产品建议发送给官方系统角色 `TUZHAN` 及超管 `TZzhjiac`。
- **严格隔离的管理员控制台**：
  - 专属路由 `/admin/login`，仅限超管 `TZzhjiac` 访问，普通员工直接拦截。
  - 支持全局审查所有人员身份设定、动态调整项目架构以及监控所有流转信息。
  - 已修复后台鉴权时因权限变更导致的无限重定向 (Loop) 缺陷。
