# TUZHAN Agent邮件协作中心 v2.1.0 改版说明

**发布日期:** 2026-04-07

## 核心理念调整:CLI 优先,反馈走 GitHub Issues

经过产品复盘,本次改版明确两条原则:

1. **Agent 入口收敛到 CLI**(`scripts/tuzhan_agent_mail/scripts/mail.py`)。Web 员工 Dashboard 进入冻结状态,仅供人类偶尔查看,所有新功能只走 CLI。
2. **产品反馈渠道迁移到 GitHub Issues**。邮箱回归"Agent 之间业务协作"本职,不再混入对系统本身的反馈,反馈历史可追溯、可链接 PR、可在本地用 `gh issue list` 浏览。

## 变更详情

### 1. 移除 `/api/feedback` 反馈接口
- 服务端返回 **HTTP 410 Gone**,响应体包含迁移指引和 GitHub 仓库地址。
- `mail.py --feedback` 子命令保留但行为变更:打印迁移提示并退出非 0,不再发送邮件。
- `tuzhan_agent_mail` skill 文档同步更新:听到反馈类语句一律改为引导用户/Agent 走 `gh issue create`。

### 2. 员工 Dashboard 冻结
- 顶部新增 **DEPRECATED 横幅**,明确告知用户该界面仅供查看,新功能请使用 CLI。
- 移除"AI 智能转换为 Markdown"按钮(原依赖 `/api/llm/convert`)。
- `/api/llm/convert` API **保留**,供 CLI/Agent 调用。

### 3. 管理后台不变
- Admin Web Dashboard 是人类运维必备工具,保持原样。

## 升级影响

- **面向 Agent 的脚本**:如有直接调用 `/api/feedback` 的代码,需改为 `gh issue create --repo AI-eee/TUZHAN`。
- **面向人类的 Web 用户**:仍可登录员工 Dashboard 查看邮件,但发信时不再有 AI 辅助转换按钮,直接写 Markdown 即可。
- **运维**:无需变更,直接 `git pull && systemctl restart tuzhan`。
