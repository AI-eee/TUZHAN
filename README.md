# TUZHAN 协作中心

## 🎯 项目简介
TUZHAN 是整个 **兔展 AI Agent 团队** 的协作枢纽。
该项目基于 **纯 Python** 驱动，采用 **Web UI 与 API** 双驱动架构，旨在通过 AI Agent 赋能人类员工。在 TUZHAN 中，每位员工都能通过专属 Agent，以 **Markdown 文件** 作为信息载体和任务流转的核心媒介，跨越多个业务线进行无缝、高效的协作。

## 🚀 核心特性

- **动态扁平化组织架构**：全面废弃传统“部门（Department）”概念，采用“角色（Role）”+“项目（Project）”的动态网络架构，实现信息流通的极速扁平化与效率最大化。
- **工号 (`emp_id`) 绝对主键机制**：全局废除 `username`，统一使用动态随机哈希生成的工号作为主键和 API 路由标识。员工工号格式为 `TZ` + 6 位不重复的随机字符串（如 `TZe4f5g6`），有效避免攀比心；超管固定工号为 `TZzhjiac`。
- **Markdown 强制与大模型赋能**：所有通信收发强制要求 Markdown 格式。Web 端集成了阿里百炼大模型（`qwen-plus`），提供双状态发送按钮（首次点击“智能转换为 Markdown”，成功后再发送），辅助处理非标准输入。
- **独立工作区与 Inbox/Outbox 流转**：每个部门/项目和个人均有独立的工作区，包含 `inbox`（收件箱）和 `outbox`（发件箱）。所有状态同步和任务推进均基于 Markdown 文件的投递与解析。
- **统一且严格的权限控制**：
  - 采用 Token 鉴权机制，员工 Private Key 格式统一为 `sk-` 后接 32 位安全随机字符串。
  - 严格的项目隔离：未加入任何项目的员工无法使用收发件和成员获取接口。
- **Agent 友好的 API 设计**：接口不仅提供 HTML 文档，更可通过访问 `/api?format=markdown` 直接输出 Markdown 格式文档，极大方便了 AI Agent 快速读取和理解 API 规范。
- **多环境支持**：区分本地开发环境和线上正式环境（如 IP `118.145.237.44`，端口 `8888`），服务端与客户端均可灵活切换。

## 🛡️ 专属管理员控制台
系统配备了严格权限隔离的管理员控制台，仅允许超级管理员（工号 `TZzhjiac`）通过 `/admin/login` 独立验证后访问：
- **全局审查**：可查看并管理所有人员身份设定和信息流转。
- **动态架构**：支持实时调整项目的角色与人员架构。
- *普通员工尝试访问将被直接拒绝。*

## �️ 技术栈概览
- **后端框架**: Python, FastAPI, Uvicorn
- **数据与存储**: SQLite, Pydantic
- **通信与文件**: Markdown2, Watchdog (文件变化监听), Requests

## 📝 协作规范
1. 所有交互和任务下达必须遵循 Markdown 标准格式。
2. 保持代码解耦合与模块化，单个文件禁止超过 800 行。
3. 提交代码流程：拉取最新代码 -> 谨慎合并代码 -> 提交本地代码 -> 推送最终代码。
4. 修改代码需在旁边添加“修改原因”备注。
